# dev-knowledge-base

> 一个人的开发结界展开中。
> 把踩过的坑、修过的 bug、配过的环境，慢慢封印成以后随时可翻的知识卷轴。

这是一个偏个人向的开发知识库。

这里主要记录：

- 开发环境配置
- 工具链使用经验
- 常见问题排查过程
- 已验证可复用的解决方案

目标很简单：

- 少重复踩坑
- 少临时百度
- 少在“我之前明明解决过”里反复轮回

## 目录建议

```text
dev-knowledge-base/
  README.md
  docs/
    ai-tools/
    dev-env/
    git/
    network/
    terminal/
    troubleshooting/
    windows/
```

目前先从最常用、最容易复用的内容开始积累，后面再慢慢扩展成完整的知识图鉴。

## 收录范围

- 开发环境配置和坑位
- 常见排障结论
- 工具链使用经验
- 可复用的命令与验证步骤

## 记录原则

- 先写成文档，优先解决“下次自己能不能快速看懂”
- 结论尽量写完整，最好做到复制命令就能复现
- 一篇文档只解决一个核心问题，避免越写越散

## 什么时候再进化成 skill

当某个问题开始重复出现，而且你希望 Codex 以后能按固定流程自动执行排查或处理时，再把它从“知识卷轴”升级成 skill 会更合适。

换句话说：

- 一次性结论，先写文档
- 高频重复动作，再做 skill
